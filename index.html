Introduction

This project involves setting up a Windows 11 virtual machine and a Kali Linux virtual machine on a Mac using UTM. The goal is to practice penetration testing by attempting to exploit vulnerabilities in the Windows VM using tools from the Kali VM.

Setting Up the Virtual Machines

1. Installing UTM

  1. Download UTM: Visit the UTM website and download the latest version for macOS. Install UTM by dragging the application into your Applications folder.
  2. Launch UTM: Open UTM from your Applications folder.

2. Setting Up the Windows 11 VM

  1. Download a Windows 11 ISO: Go to the official Microsoft Windows 11 download page and download the ISO file.
  2. Create a New Virtual Machine in UTM: Click the + button in UTM to create a new virtual machine. Select Virtualize → Windows and click Next.
  3. Configure the VM: Give your VM a name like Windows 11. Allocate enough resources (e.g., at least 4 GB of RAM and 2 CPU cores). Click Browse to select the Windows 11 ISO. Ensure UEFI is selected.
  4. Network Configuration: Under the Network tab, choose a network interface that connects the Windows VM to the same network as your Kali Linux VM. Use Shared Network (NAT) or a custom network.
  5. Finalize and Install Windows 11: Click Save and then Play to start the VM. Follow the on-screen instructions to install Windows 11.

3. Setting Up the Kali Linux VM

  1. Download the Kali Linux ISO: Visit the Kali Linux download page and download the ISO file.
  2. Create a New Virtual Machine in UTM: Click the + button to create another virtual machine. Select Virtualize → Linux/Other and click Next.
  3. Configure the Kali VM: Name this VM Kali Linux. Allocate resources similar to the Windows VM. Select the Kali Linux ISO file.
  4. Network Configuration: Ensure that the network settings match those of the Windows VM to allow both machines to communicate.
  5. Finalize and Install Kali Linux: Save the VM configuration and start it. Follow the installation steps for Kali Linux.
  6. Update and Install Tools: Once Kali Linux is installed, update the system and install any additional tools you need: 
    sudo apt update && sudo apt upgrade -y
sudo apt install -y metasploit-framework nikto dirb

4. Networking Between VMs

  1. Check Network Connectivity: Open a terminal in Kali Linux and ping the Windows VM to ensure they can communicate: 
    ping <Windows_VM_IP>
  2. Troubleshooting: If the VMs can’t communicate, check the network settings in UTM and make sure both VMs are connected to the same network interface.

5. Setting Up Monitoring Tools on Windows 11

  1. Install Wireshark: Download Wireshark from wireshark.org and install it on the Windows 11 VM.
  2. Configure Windows Defender Firewall: Ensure that the firewall is configured to allow traffic between the VMs, especially for the ports and services you intend to scan or exploit.
  3. Enable Remote Desktop (Optional): If you want to manage the Windows VM remotely from Kali, enable Remote Desktop on Windows 11.

Conducting the Penetration Test

With the VMs set up and connected, you can now proceed with penetration testing.

1. Reconnaissance

From the Kali VM, use tools like nmap to scan the Windows VM for open ports and services.

2. Exploitation

Attempt to exploit discovered vulnerabilities using tools like Metasploit.

3. Analysis

Use Wireshark on the Windows VM to capture and analyze the traffic during the penetration test.

Conclusion

This project provided hands-on experience with setting up virtual machines, networking, and conducting basic penetration testing. The detailed steps and challenges encountered helped reinforce concepts in cybersecurity and systems administration.
